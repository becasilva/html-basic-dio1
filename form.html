Formulário







OBS: A TAG select não tem compatibilidade com o navegador Edge da Microsoft.

O atributo multiple permite que mais de uma opção da lista sejam selecionados.

Sobre formulários:

Metódo de envio de dados:

- Metódo GET é inseguro, pois exibe informações, inclusive senhas, na URL.
- Método POST envia uma requisição ao servidor, de maneira a escapsular os dados.

Método POST é bem mais seguro!

Target

Onde abrir o documento linkado. Por ex, se será aberto na mesma página ou em uma nova guia.

  • _blank: Para abrir em uma nova aba.
  • _self: Para abrir na mesma aba. (Em si mesma) [Padrão]

Autocomplete

Permite salvar informações e utilizar elas novamente, se estiver como ON e não permitirá se estiver como OFF.
Além de ser possível, trafegando entre abas, voltando pra o formulário, os dados permanecem.

On: Evento

É uma ação.

Vamos usar muito o "onsubmit", utilizando JavaScript.
Ou seja, que ação será realizada ao apertar o botão de enviar.

É muito usado para verificação e validação!

EX: Se a idade é válida.

Input

Definição: Caixa de preenchimento de valores.


→ Pode-se utilizar min="valor" e max="valor" para definir um valor mínimo e máximo possível.
tep="valor a ser pulado", permite pular os valores de 5 em 5 por exemplo.

→ Button type="button", "Reset", "Submit"

→ Também se usa "min=0", max="90" e value="0" como valor padrão inicial


→ No input type="email", o navegador só permite enviar o form, se estiver em formato correto.

→ No input type="url", o navegador só permite enviar o form, se estiver em formato correto.

→ No input type="date", o navegador pega o idioma do sistema para formatar a data.

→ No input type="week", não há compatibilidade em todos os navegadores.




→ Em Radio o atributo name="mesmonome" deve ser igual em todas as opções.


→ Valor invisível.


→ Por padrão, apenas um arquivo pode ser selecionado.
